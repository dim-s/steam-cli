# DECISIONS

Продуктовые развилки проекта: что открыто и ждёт решения владельца, что уже
решено и не подлежит переоткрытию. Статус работы живёт на доске `.boardown/`,
здесь — только решения.

## Открыто

### Читать ли данные ДРУГИХ игроков

Партнёрский ключ технически открывает чтение чужих библиотек и прогресса по
произвольному steamid (`IPlayerService/GetOwnedGames`,
`ISteamUserStats/GetPlayerAchievements`). В 1.5.0 это **не реализовано**
сознательно: README проекта прямо декларирует такие данные как *intentionally
out of scope*, и открытие их — разворот заявленной позиции продукта, а не
техническая мелочь. Вопрос поднят при разведке партнёрского API (июль 2026) и
ждёт решения владельца.

Против: приватность чужих аккаунтов; продукт перестаёт быть «инструментом
рыночной разведки» и становится инструментом слежки за игроками; README
придётся переписывать в части обещаний.
За: данные всё равно доступны любому владельцу ключа, а агенту иногда нужен
профиль автора отзыва глубже публичного.

## Решено (свод)

- **2026-07-18 — cookie-скрейп `partner.steamgames.com` не делаем.** Изначально
  выглядел единственным путём к вишлистам и продажам, поэтому обсуждался как
  развилка. Снят: найден официальный `IPartnerFinancialsService`
  (`GetAppWishlistReporting`, `GetDetailedSales`), который даёт те же данные
  через обычный ключ, без протухающей cookie и хрупкого парсинга HTML.
- **2026-07-18 — флага `--api-key` не будет.** Ключ, переданный аргументом,
  виден другим пользователям машины через `ps` и остаётся в истории шелла.
  Источники ключа: переменная окружения и файл `0600`, заполняемый
  `auth --set` со stdin. Прецедент `--insecure` не переносится: он ослабляет
  проверку локально, а не выносит секрет наружу.
- **2026-07-18 — один ключ, без профилей.** Владелец подтвердил: несколько
  партнёрок на одной машине маловероятны.
- **2026-07-18 — отдельной команды `stats` не будет.** У всех игр владельца
  `availableGameStats.stats` пуст; команда давала бы гарантированную пустоту.
  Статы, если они есть, показываются внутри `achievements`.
