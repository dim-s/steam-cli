Правило: после любого изменения CLI-поверхности (новая подкоманда/аргумент) в сессии, где ранее уже делался глобальный `pip install` этого пакета — переустанови глобальную копию (`pip install --force-reinstall .` / эквивалент для pipx/uv) И проверь именно вызовом РЕАЛЬНОГО установленного бинаря `steam-cli <новая-команда> ...` из директории ВНЕ воркчерства репозитория. Прогон через `python -c "import steam_cli"` или из cwd репозитория не считается проверкой — он резолвится на локальный чекаут, а не на установленную копию, и не ловит рассинхрон.
Почему: механизм промаха — промежуточная переустановка в начале сессии создала ложное ощущение «глобально всё синхронно»; функционал, дописанный ПОСЛЕ неё в той же сессии, молча остался недоступен пользователю до живого прогона из другого cwd. Владелец поймал это лично, до того как аудит менеджера подтвердил механизм.
